# Review 报告修改（MiniMax 模型）

> **路径规则**：所有文档路径以 `smartroute.config.json` 中的 `documents` 配置为准，不要使用硬编码路径。

$ARGUMENTS

## 第一步：读取配置
1. 读取 `smartroute.config.json` 中 `documents.review_dir` 确定 Review 报告目录

## 第二步：修复流程
1. 读取 Review 报告目录下最新日期的 review-report-{日期}.md
2. 按严重程度排序处理：严重 → 中等 → 轻微
3. 逐条修复，每修复一项后：
   - 确保编译通过
   - 在报告中标注修复状态（已修复 / 暂不修复 + 原因）

## 修复规范
1. 每次修改只涉及一个 Review 问题，避免交叉修改
2. 不要大范围重构，只修复 Review 报告中明确指出的问题
3. 如果某个问题的修复方案不确定，标注"需确认"而不是猜测修改
4. 修改代码时保持现有的注释规范（中文注释 + 函数头注释）

## 第三步：回归验证
1. 所有修复完成后，执行完整编译
2. 运行系统测试和单元测试，确认无新增失败
3. 如有新增失败，优先修复

## 第四步：输出
1. 更新 Review 报告中每条问题的修复状态
2. 输出修复摘要到 .pipeline/review-fix-summary.md，包含：
   - 修复数量统计（已修复 / 暂不修复）
   - 回归测试结果

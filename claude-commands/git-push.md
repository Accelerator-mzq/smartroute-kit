# 上传 Git（MiniMax 模型）

请执行以下 Git 操作：

## 提交前检查
1. 确认当前处于正确的分支（如不确定，先执行 `git branch` 确认）
2. 执行 `git diff --stat` 查看修改的文件列表
3. 检查是否有不应提交的文件（临时文件、日志文件、.env 等敏感文件）
4. 确认 .gitignore 已正确配置

## 提交流程
1. `git add` 所有修改的文件（排除不应提交的文件）
2. 生成 commit message（中文），格式：
   ```
   [类型] 简要描述

   - 具体修改点 1
   - 具体修改点 2
   ```
   类型包括：[功能] [修复] [重构] [测试] [文档] [配置]
3. `git commit`
4. `git push`

## 异常处理
- 如果 push 失败（如远程有新提交），先 `git pull --rebase` 再重试
- 如果有合并冲突，列出冲突文件并等待用户指示
- 不要强制 push（禁止 `git push -f`）
